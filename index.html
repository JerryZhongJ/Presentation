Configuration Test

Introduction

  • Configuration Testing is the process of testing the system under each configuration of the supported software and hardware.
  • Includes: Hardware & Software
    • various OS, various browsers, various supported drivers, distinct memory sizes, different hard drive types, various types of CPU
  • To:
    • Determine if it fulfills the configurability requirements
    • Find out optimal configurations
    • Identify the defects
    • Analyze the system efficiency and performance

An Example

  • An app: server + datebase + client
  • Client: Windows 10, Windows 7, Windows XP, Linux.
  • Server: Ubuntu Server, Windows Server 2016, Novell Open Enterprise Server.
  • Database: Microsoft SQL Server, IBM DB2, MySQL.
  • 36 combinations in total

How

  • Basic Configuration Matrix: consists of various combinations of software and hardware configurations
  • Priority: Prioritizing the configurations as its difficult to test all the configurations
